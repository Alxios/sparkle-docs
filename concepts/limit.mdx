Sparkle offre une puissante plateforme de création de bots Twitch, mais il existe certaines limites à prendre en compte lors de l'utilisation de l'application.

Ces limites sont conçues pour garantir une utilisation équitable des ressources et une expérience fluide pour tous les utilisateurs.

### Limites d'exécution

Est considéré comme une exécution, tout code qui est exécuté dans la runtime de Sparkle. Cela inclut les commandes, les tasks, les schedules et les hooks.

- **Durée d'exécution maximale** : Les exécutions dans la runtime sont limitées à 5 secondes. Tout code qui dépasse cette limite sera automatiquement interrompu.

- **Limite de mémoire** : Chaque exécution de code est limitée à 8 Mo de mémoire. Assurez-vous que votre code reste dans cette limite pour garantir des performances optimales.

- **Limite de concurrence** : Chaque chaine est limité à **1 exécutions simultanées** pour le plan gratuit et **5 pour le plan PRO**.

- **Sous Tache** : Lorsque vous utiliser la fonction `invoke` ou `invokeUntil` vous créer une sous tache. Vous êtes limitez à 100 sous tache par exécution.

<Info>

  Twitter, pour établir un plan d'abonnement personnalisé.
</Info>

### Limites du tchat Twitch

- Les envois de message dans le tchat Twitch sont limités à 5 tous les 30 secondes. Cette limite est en place pour éviter les abus et s'acorder avec les limites de Twitch.
